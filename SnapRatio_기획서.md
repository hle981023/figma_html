# SnapRatio — Illustrator 사이즈 정리 에이전트 기획서

> 초안 v0.1 · 2026-07-01 · 작성: 보혜 + Claude

## 1. 한 줄 정의

일러스트레이터에서 자유롭게 그린 오브젝트의 **소수점 사이즈**를, **원래 가로세로 비율을 유지한 채** 소수점 없는 정수 사이즈로 정리해주고, 그 과정에서 가까운 **정수비·배수를 추천**하며, 현재 그래픽이 **몇 : 몇 비율의 그리드**인지 화면으로 보여주는 UXP 플러그인 에이전트.

---

## 2. 문제 (Why)

그래픽·로고·인쇄물 작업은 보통 "먼저 감으로 그리고, 나중에 수치를 맞추는" 순서로 진행된다. 이 과정에서 오브젝트 크기가 `127.34 × 84.62 mm`처럼 소수점으로 떨어지는 일이 반복된다.

- 소수점 사이즈는 재단·조판·개발 핸드오프에서 오차와 커뮤니케이션 비용을 만든다.
- 손으로 정수로 바꾸면 **비율이 깨진다** (가로만 127로 반올림하면 세로는 그대로라 형태가 미세하게 틀어짐).
- "그럼 어떤 깔끔한 숫자로 맞춰야 원본과 가장 가깝고 비율도 유지되나?"를 매번 암산/시행착오로 해결하고 있다.

**핵심 통찰:** 이건 단순 반올림이 아니라 *"현재 비율에 가장 가까운 단순 정수비를 찾고, 그 비율의 정수 배수 중 원본에 가장 가까운 값을 고르는"* 수학 문제다. 사람이 매번 하기엔 번거롭지만 자동화엔 딱 맞는다.

| 유저 | 상황 | 니즈 |
|---|---|---|
| 그래픽 디자이너 | 로고·심볼을 자유 드로잉 후 규격화 | 비율 유지한 정수 사이즈 |
| 편집·인쇄 디자이너 | 인쇄물 최종 단계에서 mm 정리 | 단위별(mm) 깔끔한 수치 |
| 브랜딩 디자이너 | 로고 그리드/비율 규정 문서화 | 몇:몇 비율인지 시각적 증명 |

## 4. 기능 (What)

### 4.1 소수점 정리 — 비율 유지 정수 스냅 *(핵심)*
- 선택 오브젝트의 현재 W·H를 읽어 **비율 R = W/H** 계산.
- R에 가장 가까운 **단순 정수비 p:q**를 찾음 (연분수/Stern-Brocot 기반, 분모가 작을수록 = 단순할수록 우선).
- p:q의 정수 배수 `(p·k, q·k)` 중 원본에 가장 가까운 후보 3~4개를 **편차%와 함께** 제시.
- 기본 앵커 = **좌상단 고정** (재단선·조판 기준에 맞음).
- 결과는 항상 정수 → 소수점 원천 차단.

**예시 (검증 완료)**

| 입력 | 현재비율 | 추천 정수비 | 1순위 결과 | 편차 |
|---|---|---|---|---|
| 127.34 × 84.62 | 1.505 | 3:2 | **126 × 84** | 1.05% |
| 159.2 × 99.6 | 1.598 | 8:5 | **160 × 100** | 0.50% |
| 63.51 × 47.63 | 1.333 | 4:3 | **64 × 48** | 0.78% |
| 420.1 × 296.9 | 1.415 | 7:5 | **420 × 300** | 1.04% |

### 4.2 비율·배수 추천 및 변형
- 각 추천 카드에 **비율(3:2)·정수 사이즈·원본 대비 편차·그리드 칸 수** 표시.
- 카드 선택 후 **배수 버튼(×0.5, ×2, ×3)**으로 즉시 확대/축소 — 정수비×정수 구조라 배수를 곱해도 항상 정수 유지.
- "원복" 버튼으로 원본 측정값 복귀.

### 4.3 그리드 뷰 — 비율 시각화
- 선택한 정수비 p:q에 맞춰 **p열 × q행의 그리드**를 오브젝트 끝선에 정확히 오버레이.
- 각 칸은 정사각형이 되어(정수비 유지 시) "이 그래픽은 3:2다"를 눈으로 증명.
- 각 변에 실제 치수 + 칸 수 라벨, 중앙에 `p : q` 라벨 표시.
- 좌상단 앵커 위치를 마커로 표시.

### 4.4 단위 선택
- **mm / pt / px / in** 토글 버튼. 작업 성격마다 기준 단위가 달라 매번 선택 가능.
- 선택 단위 기준으로 "정수"를 판정 (mm 정수 vs px 정수는 다름).

## 5. UX 플로우

```
오브젝트 선택
   ↓
[에이전트가 W·H·비율 자동 측정]
   ↓
단위 선택 (mm/pt/px/in)
   ↓
비율 유지 정수 후보 3~4개 카드 제시 (편차 순)
   ↓
카드 선택 → 그리드 뷰에 실시간 반영 (몇:몇 확인)
   ↓
(선택) 배수 변형
   ↓
[적용] → 좌상단 고정한 채 오브젝트 리사이즈
```

## 6. 핵심 로직 명세

```
입력: W, H (현재 측정값), unit
1. R = W / H
2. p:q = simplestRatio(R)
   - q를 1..40까지 오름차순, p = round(R·q)
   - |p/q − R| / R ≤ tol(=2%) 인 첫 (p,q)를 약분하여 반환 (가장 단순)
3. 후보 생성: k ≈ (W/p + H/q)/2 주변의 정수 k들
   - 각 후보 (p·k, q·k), 편차 = max(|w−W|/W, |h−H|/H)
4. 편차 오름차순 정렬 → 상위 4개, 1순위를 '추천'으로 강조
5. 적용: 좌상단(x,y) 고정, W→w, H→h 로 스케일
```

- **비율 유지가 최우선**이므로 w/h는 항상 정확히 p:q. 소수점은 구조적으로 발생하지 않음.
- `tol`(허용 오차)은 사용자 설정으로 노출 가능 (엄격 1% ↔ 관대 3%).

## 7. 엣지 케이스 / 열린 질문

1. **무리비(A 규격 등):** A3·A4 용지는 비율이 √2 ≈ 1.414로 무리수라 완벽한 정수비가 없다(7:5로 근사됨). → *"표준 규격(ISO A, 황금비 등) 스냅 모드"*를 별도 옵션으로 둘지 논의 필요.
2. **정사각형 근처:** 1.006 → 1:1 로 스냅. 의도한 미세 직사각형이면 오히려 방해일 수 있음 → tol 조절 or "비율 잠금" 토글.
3. **stroke:** 이번 범위에서는 **무시**(기하학적 bounds 기준). 최종 단계에서 필요 시 "보이는 크기(visual bounds) 기준" 옵션 추가.
4. **다중 선택:** 여러 오브젝트 동시 정리 시 개별 정리 vs 그룹 bounds 기준 — 추후.
5. **회전된 오브젝트:** 회전 상태의 bounds 처리 정의 필요.
6. **앵커 옵션 확장:** 현재 좌상단 고정. 중심/우하단 등 선택지 추후 제공.

## 8. 기술 형태

- **UXP 플러그인 패널** (Illustrator). 패널 UI = 본 목업의 사이드바 + 그리드 뷰.
- 오브젝트 측정·리사이즈: UXP DOM API (`selection`, `width/height`, `translate`).
- 추천 로직: 순수 JS (외부 의존성 없음, 목업의 알고리즘 그대로 이식 가능).
- 그리드 오버레이: 패널 내 SVG 프리뷰(경량) + 옵션으로 아트보드 위 가이드 생성.
